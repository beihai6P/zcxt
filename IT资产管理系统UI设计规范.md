IT资产管理系统UI设计规范
1. 规范总则
1.1 设计目标
本规范旨在统一IT资产管理系统的UI设计风格、交互逻辑与视觉标准，确保系统界面简洁直观、操作高效流畅，降低用户学习成本；同时兼顾专业性与易用性，适配管理员、普通用户、管理层等不同角色的使用需求，保障多场景下的操作一致性，提升用户体验与系统可用性，助力资产全生命周期管理高效落地。
1.2 适用范围
本规范适用于IT资产管理系统所有UI相关设计，包括PC端网页界面（核心操作端）、手机端浏览器适配界面（扫码查询、简单申请）、数据大屏界面，涵盖页面布局、组件设计、视觉风格、交互逻辑、响应式适配等所有UI相关环节，指导设计、开发、测试全流程的UI落地工作。
1.3 设计原则
?? 一致性原则：同一功能的操作逻辑、视觉样式（按钮、输入框、图标等）保持统一，跨页面、跨模块不出现差异，降低用户记忆成本。
?? 易用性原则：操作流程简洁，减少不必要的步骤，关键功能（资产录入、扫码查询、审批）突出展示；报错提示清晰，引导用户快速解决问题。
?? 专业性原则：贴合IT资产管理场景，视觉风格简洁、严谨，避免冗余装饰，重点突出资产数据、操作功能，符合企业办公系统的专业调性。
?? 可扩展性原则：UI设计预留拓展空间，支持后续新增资产类型、功能模块时，无需大规模调整整体风格，确保系统迭代的连贯性。
?? accessibility原则：界面配色对比清晰，支持字体大小调整，操作按钮可点击区域适中，适配不同用户的使用习惯，兼顾易用性与包容性。
2. 整体视觉风格
2.1 配色规范
2.1.1 主色调
核心主色：#1E88E5（蓝色），用于系统主标题、导航栏、核心按钮、重点强调内容，体现专业、严谨、高效的调性，契合企业办公系统的定位，同时传递可靠、安全的视觉感受（适配资产数据管理的安全需求）。
辅助主色：#64B5F6（浅蓝），用于次级按钮、 hover 状态、进度条、提示框等，与主色形成层次对比，提升界面活力，同时保持视觉统一性。
2.1.2 功能色
?? 成功色：#4CAF50（绿色），用于操作成功提示、审批通过、正常状态标识（如在用资产状态）。
?? 警告色：#FFC107（黄色），用于预警提示、待审批状态、盘盈/盘亏标记、库存不足提醒。
?? 错误色：#F44336（红色），用于操作失败提示、驳回状态、报废资产标识、越权操作警告。
?? 中性色：#333333（深灰）用于正文文字；#666666（中灰）用于次要文字、提示文字；#EEEEEE（浅灰）用于背景色、分割线；#FFFFFF（白色）用于卡片、面板背景，提升界面清晰度。
2.1.3 配色禁忌
?? 禁止使用高饱和度、对比度过强的配色，避免视觉疲劳（如纯红+纯绿、亮黄+亮紫）。
?? 同一页面内配色不超过4种（主色+辅助色+1-2种功能色），避免色彩杂乱，影响信息读取。
?? 文字与背景配色需保证足够对比度（至少4.5:1），确保弱视力用户可清晰读取内容，禁止浅色文字搭配浅色背景（如#999999文字搭配#EEEEEE背景）。
2.2 字体规范
2.2.1 字体选择
全局统一使用无衬线字体，优先适配系统默认字体，确保跨设备、跨浏览器显示一致：
?? PC端：微软雅黑（Microsoft YaHei）、思源黑体（Source Han Sans CN）， fallback 字体为 Arial。
?? 移动端：思源黑体（Source Han Sans CN）、San Francisco（iOS）、Roboto（Android），确保扫码查询、简单操作时的字体清晰度。

2.2.3 字体样式规范
?? 标题文字统一使用深灰色（#333333），正文文字使用中灰色（#666666），辅助文字使用浅灰色（#999999）。
?? 禁止使用斜体字体（特殊强调除外，如异常提示），避免使用下划线（仅链接可使用，且链接颜色统一为#1E88E5）。
?? 表格内容、表单输入文字左对齐，标题文字左对齐（数据大屏标题可居中），确保视觉整齐。
2.3 图标规范
2.3.1 图标风格
全局采用线性图标，线条粗细统一为2px，风格简洁、扁平化，避免复杂装饰，确保图标含义清晰、辨识度高，贴合IT资产管理场景（如资产、二维码、审批、备份等相关图标）。
2.3.2 图标尺寸
?? 导航栏图标：24px×24px（如首页、资产管理、系统管理等导航图标）。
?? 操作按钮图标：20px×20px（如新增、删除、编辑、导出、扫码等操作图标）。
?? 辅助图标：16px×16px（如提示图标、状态图标、表格内操作图标）。
?? 数据大屏图标：32px×32px（核心指标图标、预警图标），可根据大屏布局适当调整大小，保持比例一致。
2.3.3 图标颜色与使用规范
?? 默认状态：图标颜色为#666666（中灰）；hover/激活状态：图标颜色为#1E88E5（主色）；禁用状态：图标颜色为#CCCCCC（浅灰）。
?? 功能类图标（如成功、警告、错误）需与功能色对应（成功图标为绿色、警告为黄色、错误为红色）。
?? 同一功能的图标保持统一（如“新增”统一使用“+”图标，“导出”统一使用“下载”图标），避免同一功能使用不同图标导致用户混淆。
?? 图标与文字搭配时，图标位于文字左侧，间距为8px，确保视觉协调。
3. 页面布局规范
3.1 整体布局结构
系统整体采用“顶部导航栏+左侧侧边栏+中间内容区”的经典办公系统布局，适配PC端操作，移动端适配为“顶部导航+底部操作栏”，确保操作便捷性。
3.1.1 PC端布局（核心布局）
?? 顶部导航栏：高度60px，包含系统logo、当前登录用户信息（头像、用户名、退出按钮）、全局搜索框、通知提醒图标（预警、审批通知），背景色为白色，底部添加1px浅灰色分割线。
?? 左侧侧边栏：宽度220px，背景色为#FFFFFF，包含系统所有功能模块导航（资产基础管理、历史追溯、大数据可视化、AI辅助、系统管理），导航项分为一级导航和二级导航，选中项背景色为#E3F2FD（浅蓝），文字颜色为#1E88E5（主色）；折叠状态下宽度为60px，仅显示导航图标。
?? 中间内容区：自适应宽度，默认边距为20px（上下左右），背景色为#F5F7FA（浅灰），包含页面标题、操作按钮区、内容展示区（表格、表单、卡片、图表等），内容区采用卡片式布局，卡片背景色为白色，圆角8px，阴影为0 2px 8px rgba(0,0,0,0.08)。
3.1.2 移动端布局（适配手机浏览器）
?? 顶部导航栏：高度50px，包含返回按钮、页面标题、扫码按钮（核心功能），背景色为#1E88E5（主色），文字颜色为白色。
?? 内容区：自适应手机屏幕宽度，边距为15px，优先展示核心功能（扫码查询、资产详情、申请提交），简化操作流程，避免多步骤跳转。
?? 底部操作栏：高度50px，包含常用功能图标（首页、资产查询、我的申请），背景色为白色，图标颜色默认#666666，选中状态为#1E88E5。
3.1.3 数据大屏布局
采用网格布局，分为顶部标题区、核心指标区、图表展示区、预警提示区，整体背景色为#1A1A2E（深灰蓝），提升数据可视化的视觉冲击力；各区域之间间距为20px，卡片采用圆角8px，无明显阴影，突出数据展示。
3.2 页面间距规范
?? 页面内大间距：20px（如内容区与侧边栏、顶部导航栏的间距，卡片之间的间距）。
?? 中等间距：16px（如页面标题与操作按钮区的间距，表单内不同分组的间距）。
?? 小间距：8px（如按钮之间的间距，图标与文字的间距，表单标签与输入框的间距）。
?? 细微间距：4px（如文字与边框的间距，提示语与输入框的间距）。
3.3 页面标题与操作区规范
?? 页面标题：位于内容区顶部左侧，使用二级标题样式（18px，#333333），下方添加1px浅灰色分割线，分割线宽度与标题对齐。
?? 操作按钮区：位于页面标题右侧，按钮排列从左至右依次为“新增/提交”“导出”“刷新”“筛选”等，按钮之间间距为8px，优先展示核心操作按钮，次要操作可放入下拉菜单（如“更多操作”）。

?? 按钮文字简洁明了，字数控制在4-8字（如“新增资产”“提交审批”），避免冗长。
?? 同一页面内按钮样式统一，核心操作使用主按钮，且数量不超过2个，避免多个主按钮导致视觉混乱。

?? 按钮可点击区域不小于48px×36px，确保点击精准，避免误操作。
4.2 表单组件
4.2.1 输入框
?? 样式：高度36px，边框1px solid #DDDDDD，圆角4px， padding 0 12px；focus状态边框颜色#1E88E5，添加2px solid #E3F2FD的外边框；禁用状态背景色#F5F5F5，边框颜色#CCCCCC，文字颜色#999999。
?? 占位符：文字颜色#999999，字体大小14px，内容简洁，提示用户需输入的信息（如“请输入资产名称”“请选择采购日期”）。
?? 报错状态：边框颜色#F44336，右侧添加错误图标（16px×16px），下方显示报错提示（12px，#F44336），提示内容具体（如“采购日期不可为空”“资产型号格式错误”）。
?? 适配场景：单行输入（如资产名称、型号）使用普通输入框；多行输入（如备注、故障描述）使用文本域，高度默认80px，可拖动调整大小。
4.2.2 下拉选择框
?? 样式：与输入框保持一致（高度36px，边框1px solid #DDDDDD，圆角4px），右侧添加下拉箭头（16px×16px），颜色#666666；focus状态与输入框一致。
?? 下拉菜单：背景色白色，边框1px solid #DDDDDD，圆角4px，阴影0 2px 8px rgba(0,0,0,0.08)；选项高度32px，hover状态背景色#E3F2FD，文字颜色#1E88E5；选中项背景色#E3F2FD，文字颜色#1E88E5。
?? 使用规范：选项较多时（如部门选择、资产类型选择），添加搜索功能；支持多选（如权限分配），多选时显示已选选项标签，标签可删除。
4.2.3 日期选择器
?? 样式：与输入框保持一致，右侧添加日历图标（16px×16px），颜色#666666；选中日期背景色#1E88E5，文字颜色白色；hover日期背景色#E3F2FD。
?? 适配场景：单个日期选择（如采购日期）使用普通日期选择器；日期范围选择（如盘点时间范围、日志查询时间范围）使用双日期选择器，两个输入框之间添加“至”字分隔。
4.2.4 表单布局规范
?? 表单采用网格布局，每行2-3个表单项（根据页面宽度调整），表单项之间间距为16px（左右）、20px（上下）。
?? 表单标签：右对齐，宽度统一为100px，字体大小14px，颜色#333333，与输入框间距为8px。
?? 必填项：标签右侧添加红色“*”标识，位置在标签文字右侧，间距4px。
?? 表单底部操作区：包含“提交”“取消”按钮，按钮右对齐，间距8px；表单较长时，添加“回到顶部”按钮，固定在表单右侧。
4.3 表格组件
4.3.1 表格样式
?? 整体样式：表格宽度100%，边框-collapse: collapse；表头背景色#F5F7FA，文字颜色#333333，字体大小14px，高度40px；表体行高36px，文字颜色#666666，奇数行背景色#FFFFFF，偶数行背景色#F9FAFB（浅灰），提升可读性。
?? 边框：表头底部添加2px solid #1E88E5，表体每行底部添加1px solid #EEEEEE；无左右边框，避免边框过多导致视觉杂乱。
?? 排序：表头可点击排序（升序/降序），点击后添加排序图标（箭头），排序状态下表头文字颜色#1E88E5。
4.3.2 表格操作规范
?? 操作列：位于表格最右侧，宽度120px，操作按钮采用文本按钮，排列方式为横向（间距8px），常用操作（查看、编辑）在前，危险操作（删除）在后。
?? 分页：表格底部添加分页组件，居中对齐，包含页码、上一页、下一页、跳转页码输入框、每页条数选择器；每页条数默认10条，可选择20条、50条、100条。
?? 筛选：表格顶部添加筛选区，筛选条件与表单组件一致，筛选按钮采用次级按钮，筛选结果实时更新；支持高级筛选（点击“高级筛选”展开更多筛选条件）。
?? 批量操作：表格左侧添加复选框，可批量勾选数据，顶部操作区显示“批量删除”“批量导出”等批量操作按钮（仅管理员可见）。
4.4 弹窗组件
4.4.1 弹窗样式
?? 整体样式：弹窗居中显示，背景色白色，圆角8px，阴影0 4px 16px rgba(0,0,0,0.12)；弹窗宽度根据内容调整，默认宽度500px，最小宽度300px，最大宽度800px。
?? 弹窗头部：高度50px，包含弹窗标题（16px，#333333）、关闭按钮（16px×16px，图标颜色#666666，hover颜色#F44336），头部底部添加1px solid #EEEEEE。
?? 弹窗内容区：padding 20px，内容根据场景调整（表单、提示信息、详情展示），文字颜色#666666，字体大小14px。
?? 弹窗底部：高度50px，包含操作按钮，右对齐，按钮之间间距8px；底部顶部添加1px solid #EEEEEE，背景色#F5F7FA。
?? 遮罩层：背景色rgba(0,0,0,0.5)，点击遮罩层可关闭弹窗（确认弹窗除外）。
4.4.2 弹窗类型与适用场景
?? 确认弹窗：用于危险操作、重要操作的二次确认（如删除、报废、清理数据），包含“确认”（危险按钮）和“取消”（次级按钮），弹窗标题添加警告图标，提示语明确操作风险。
?? 表单弹窗：用于新增、编辑数据（如新增资产、修改用户信息），内容区为表单组件，底部包含“提交”（主按钮）和“取消”（次级按钮）。
?? 详情弹窗：用于查看详细信息（如资产详情、日志详情），内容区为卡片式布局，展示详细数据，底部仅包含“关闭”按钮（次级按钮）。
?? 提示弹窗：用于操作结果提示（如“资产录入成功”“审批驳回”），包含提示图标（成功/错误/警告）、提示文字，底部包含“确定”按钮（主按钮），弹窗自动关闭时间可设置（默认3秒）。
4.5 数据可视化组件
4.5.1 图表样式规范
